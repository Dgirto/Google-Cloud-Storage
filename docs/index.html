¿Qué es y para qué sirve?

El conector Google Cloud Storage permite que el agente de Ruvic gestione objetos en un bucket específico de GCS. Una vez configurado, puedes pedirle al agente cosas como:

  • “Sube este reporte al bucket con el nombre reportes/julio-2026.csv.”
  • “¿Qué archivos hay en la carpeta facturas/ del bucket?”
  • “Descarga el archivo contrato.pdf y muéstrame su contenido.”

Operaciones soportadas:

Operación Qué hace
gcs.list_objects Lista los objetos del bucket, con filtro por prefijo
gcs.upload Sube (o sobrescribe) contenido en una clave del bucket
gcs.download Descarga el contenido de un objeto
Este conector escribe gcs.upload sobrescribe el objeto de la clave indicada. El conector opera sobre un único bucket; se recomienda que el rol IAM de la cuenta de servicio esté limitado a ese bucket específico.

Prerrequisitos

  • Un proyecto de Google Cloud con un bucket de Cloud Storage ya creado.
  • Acceso a Google Cloud Console (IAM y administración) para crear una cuenta de servicio, solo durante la configuración inicial.
  • El nombre del bucket a exponer.

Configuración en el fabricante

Vas a crear una cuenta de servicio dedicada para Ruvic, con roles limitados al bucket que quieres exponer. No reutilices una cuenta de servicio de otro conector ni la de un usuario.

  1. Crea la cuenta de servicio.

    En Google Cloud Console → IAM y administración → Cuentas de servicio → Crear cuenta de servicio. No le asignes ningún rol a nivel de proyecto todavía; los roles se otorgan a nivel de bucket en el siguiente paso.

  2. Otorga los roles limitados al bucket.

    Con gsutil (o desde la consola, en el bucket → Permisos → Otorgar acceso):

    gsutil iam ch serviceAccount:ruvic-gcs@TU_PROYECTO.iam.gserviceaccount.com:roles/storage.objectViewer gs://mi-bucket-produccion
gsutil iam ch serviceAccount:ruvic-gcs@TU_PROYECTO.iam.gserviceaccount.com:roles/storage.objectCreator gs://mi-bucket-produccion
    Nota Estos roles se otorgan sobre el bucket específico (gs://mi-bucket-produccion), no sobre todo el proyecto — así la cuenta de servicio solo puede acceder a ese bucket.
  3. Descarga la clave JSON de la cuenta de servicio.

    En la cuenta de servicio → pestaña "Claves" → "Agregar clave" → "Crear clave nueva" → tipo JSON. Se descarga un archivo — su contenido completo es lo que usarás en el formulario de Ruvic.

    [ Captura: creación de clave JSON de la cuenta de servicio — reemplazar por assets/img/gcs-service-account-key.png ]
  4. Verifica el acceso desde la línea de comandos.

    Con gcloud autenticado como esa cuenta de servicio:

    gcloud auth activate-service-account --key-file=ruta/a/la/clave.json
gsutil ls gs://mi-bucket-produccion

    Si responde sin error (aunque el bucket esté vacío), el fabricante quedó listo. Anota estos valores para la siguiente sección: el contenido completo del JSON y el nombre del bucket.

Permisos requeridos

Principio de mínimo privilegio: la cuenta de servicio de Ruvic solo necesita estos roles, limitados al bucket configurado. No otorgues roles/storage.admin a nivel de proyecto ni roles/owner.

Rol Para qué sirve
roles/storage.objectViewer Listar y descargar objetos (gcs.list_objects, gcs.download)
roles/storage.objectCreator Subir objetos (gcs.upload)

Configuración en Ruvic

Entra a Settings → Conectores, busca la tarjeta Google Cloud Storage y pulsa Configurar. Diligencia el formulario con los valores que obtuviste en la sección anterior:

Campo del formulario Valor
JSON de la cuenta de servicio Pega el contenido completo del archivo descargado en el paso 3, sin modificarlo
Bucket Nombre del bucket configurado en el paso 2
Timeout de conexión Opcional; 10 segundos por defecto

Guarda y pulsa Probar conexión — la plataforma verificará el acceso al bucket configurado.

[ Captura: formulario del conector Google Cloud Storage en Settings → Conectores — reemplazar por assets/img/gcs-form.png ]

Verificación

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa al bucket 'mi-bucket-produccion'”. La tarjeta del conector cambia a estado Configurado.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele: “Sube un archivo de prueba al bucket con el texto 'hola mundo' y luego dime qué archivos hay.” El agente debe subir el objeto y listarlo de vuelta. Si lo hace, el conector quedó operativo de punta a punta.

Solución de problemas

Mensaje / síntoma Causa probable y solución
“No se pudo autenticar con la cuenta de servicio” El JSON pegado en el formulario está incompleto o corrupto. Vuelve a copiar el archivo completo del paso 3 de la sección 3, sin recortar ni modificar su contenido.
“El bucket no existe o no es accesible” Nombre de bucket mal escrito en el formulario, o la cuenta de servicio no tiene el rol otorgado sobre ese bucket. Reintenta el paso 2 de la sección 3.
El agente lista objetos pero falla al descargar uno Falta el rol storage.objectViewer, o el nombre del objeto tiene mayúsculas/espacios distintos a los mostrados.
“Google Cloud Storage no respondió correctamente” Puede ser un límite de cuota temporal de la API; reintenta en unos segundos. Si persiste, verifica que la Cloud Storage API esté habilitada en el proyecto.
El conector aparece “Sin configurar” tras guardar Algún campo obligatorio quedó vacío. Ábrelo de nuevo y completa el JSON de la cuenta de servicio y el nombre del bucket.

Preguntas frecuentes y notas de seguridad

¿El agente puede acceder a otros buckets de mi proyecto?

No, si los roles se otorgaron a nivel de bucket específico (ver sección 3), no a nivel de proyecto. Con roles.storage.objectViewer/objectCreator otorgados solo sobre gs://mi-bucket-produccion, la cuenta de servicio no puede ver ni escribir en otros buckets.

¿Dónde queda guardado el JSON de la cuenta de servicio?

Como secreto en el almacén de credenciales de la plataforma, asociado a tu usuario. La interfaz nunca vuelve a mostrarlo; solo indica “definida”. Para cambiarlo, pega uno nuevo en el formulario; para conservarlo al editar otros campos, deja el campo vacío.

Rotación de credenciales

Recomendado cada 90 días o según tu política: crea una clave nueva en Google Cloud Console (la cuenta de servicio puede tener varias claves activas), actualízala en el formulario del conector, prueba la conexión, y luego elimina la clave anterior.

¿Qué hago si las credenciales se comprometen?

  1. Elimina (o deshabilita) la clave comprometida de inmediato en Google Cloud Console → Cuentas de servicio → Claves.
  2. Revisa los Cloud Audit Logs para actividad reciente de esa cuenta de servicio.
  3. Genera una clave nueva y actualízala en el formulario del conector.

¿Expiran las credenciales?

Por defecto no (las claves de cuenta de servicio de GCP no expiran solas). Si tu organización exige rotación forzosa, puedes configurar una política de organización con antigüedad máxima de claves.

Buenas prácticas Usa siempre una cuenta de servicio dedicada con roles limitados a un único bucket; nunca uses una cuenta de usuario o una clave con roles de proyecto; no expongas buckets con datos sensibles que el agente no necesite; y evita compartir el JSON de credenciales por canales no seguros.
